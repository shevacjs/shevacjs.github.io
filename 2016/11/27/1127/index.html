weekly of 20161127

新鲜事

  • WebAssembly浏览器预览版收集社区反馈 : 可以试试哈

  • Spam Fighting @Scale 2016 : “这个里面介绍了不少公司，包括Facebook，LinkedIn, WhatsApp, Netflix 的反作弊经验和策略”, 都是视频；但是有不少感觉挺有兴趣的，包括: Whatsapp基于端到端加密是如何做spam, facebook如何用聚类的策略做假用户识别的, 同时还可以关注Networking @Scale Boston, 关于网络方面的介绍

工具技巧

  • 容易被误读的IOSTAT : 本文具体说明了await的含义, 以及在不同的业务场景下，不同磁盘类型其大概估值的含义和表针的问题(“因为await不仅包括硬盘设备处理I/O的时间，还包括了在队列中等待的时间”)

  • Keyframes: Delivering scalable, high-quality animations to mobile clients : facebook的文章，看了挺有感慨的； 初衷是支持 动态表情 展现的更友好【其实用PNG之类的也没差多少】；但是他们对这个小问题也追的很深；最后采用的是 基于『Adobe After Effects』的方案； 很多事情，看起来简单，做到极致，挑战和收获都有很多

  • pledge(2): how I learned to love web application sandboxing : pledge是OpenBDS系统下面用于控制系统调用的系统调用, 类似于linux系统下面的seccomp指令，后者可以深入了解; 在Chrome/docker等多个软件系统都有应用

系统设计

  • 关于浏览器混合内容的安全性 : 文章具体讨论了，在mixed content情况下，可以采用的安全攻击; 但是感觉博主的思路有点乱; 不过可以关注另外一个blog: Paper:安全技术精粹

  • Prometheus 和 Grafana 监控系统指南 : 挺符合我设计理念的一个监控系统；或者是信息采集系统

  • Machine Learning for Everyday Tasks : 本文具体介绍了ML 算法的一个应用, 随着机器学习算法的广泛理解和接受，相信后续有很多之前觉得复杂或者很难的事情，可以很容的被处理;

其他

  • An Interactive Tutorial on Numerical Optimization : 数值优化的一些说明和可视化展示, 先mark；

  • 理解字节序 : 很简短的文章，当做小科普吧

  • Memory management every engineer needs to know : 对于进程的内存管理的说明，还是比较精细的，包括堆，栈，静态区的位置说明, 也可以参看Anatomy of a Program in Memory

  • 服务器端人工智能，FPGA和GPU到底谁更强？ : 对GPU, FPGA等一些基础概念可以有初步的了解

  • Top Deep Learning Projects : 一些深度学习项目的list

  • Single-file public-domain/open source libraries with minimal dependencies : 一些优秀的C++ lib的合集, 先mark
